# Dashboard Overrides

The dashboard is the narrative analysis surface. It can carry more atmosphere than the schedule builder, but it still needs to stay business-facing.

## Layout

- Keep the existing wide reading frame around `1480px`.
- Use the hero first, then move immediately into widget density.
- Preserve the 12-column grid so related widgets can form clear analytical clusters.

## Priority Content

- Put cross-cutting business signal above local detail.
- Delta states, weekly cost, attendee time, top client load, and meeting count should be the fastest items to scan.
- Drilldowns must feel connected to the summary metric that opened them.

## Visual Tone

- The dashboard gets the richest use of gradient, blur, and serif display type in the app.
- Keep those effects behind the data. Decorative treatments should never interfere with chart readability.

## Interaction Rules

- Selected drilldowns need a persistent selection style, not hover-only styling.
- Tables and charts should cooperate: clicking one should clarify the other whenever possible.
- Hover can enrich, but the primary analytical state must still be visible on touch and keyboard.

## Avoid

- Promotional CTA treatment
- Empty explainer cards
- Excessively tall widgets that hide comparative context below the fold
